# personal-website

> 一个 WorkBuddy Skill — 让非技术用户用对话就能搭建个人网站，无需代码。

对话式建站：回答几个关于你是谁、你有什么内容的问题，AI 帮你选风格、搭结构、填内容、一键部署。如果需要持续更新而不碰代码，可选飞书多维表格作为 CMS 后台。

---

## 核心路径

整个工作流走的是一条「用户答 → AI 做 → 用户确认」的路：

```
身份访谈（你是谁）→ 结构确认（哪些板块、怎么排）
  → 选 CMS（纯静态 or 飞书多维表格）
  → 选风格（6 种设计，各有模板）
  → 生成内容（模板填充 + 自检）
  → 部署上线（CloudStudio）
  → 交付更新指南
```

### 关键设计决策

**Phase 1.5 是强制性步骤。** AI 不会猜你该放哪些板块——而是基于你的身份推荐一个结构，摊开给你看，等你确认了再往下走。

**默认顺序：Hero → 项目 → 文章 → 关于 → 联系。** 把访客最想看的内容（项目、文章）放前面，关于页放后面作为深度阅读者的背景信息。联系页永远在最后。

## 六种设计风格

不搞「随便选个颜色」，每种风格有完整的 CSS 系统、配色方案、字体族和布局规则。禁止混合或自定义——这不是配置器，是预制模板。

| # | 风格 | 关键词 | 适合 |
|---|------|--------|------|
| 1 | 黑白极简 | 灰度滤镜、大留白 | 设计师、工程师 |
| 2 | 可爱手绘 | emoji、圆角、暖色 | 创意自由职业者 |
| 3 | 杂志编辑 | 衬线标题、印刷品质感 | 写作者、产品人 |
| 4 | 终端极客 | 黑底绿字、命令行美学 | 有代码作品的开发者 |
| 5 | 赛博霓虹 | 青紫品红三色发光 | 技术创作者 |
| 6 | 旅程叙事 | 大地色、时间线布局 | 想讲故事的创作者 |

## 双路径 CMS

### 纯静态
内容写在 HTML 里。要更新？回来重新生成。适合每月改一次或更少的场景。零依赖、零服务器。

### 飞书多维表格 CMS
内容存在飞书表格里。在飞书上改一条记录，网站自动显示。适合每天都更新的内容型站点。支持内容状态控制（草稿/已发布）。

## 设计原则

1. **模板优先。** 不从零写 HTML。6 个成品模板是 CSS 的唯一权威来源。填内容可以，改 CSS 不行。
2. **硬约束。** 颜色不用自定义 hex——每种风格有预设调色板。字体族不可替换。响应式断点不可改。
3. **自检不可跳过。** 部署前必须过 P0 清单：meta 标签、移动端适配、图片不撑破布局、CTA 在首屏、内部锚链接有效、中文网站无英文标签。
4. **交付的不只是 URL。** 额外给一份 `HOW_TO_UPDATE.md`，告诉用户怎么改内容——无论是重新生成还是飞书表格里改。

## 边界

以下场景明确不适合：
- 电商（购物车、支付、订单管理）
- 社区/论坛（用户注册、评论、实时聊天）
- 实时协作编辑
- 复杂的权限控制
- 百万级流量的生产应用

---

## 技术栈

纯 HTML + CSS + JavaScript。单文件，零框架，零构建。部署到 CloudStudio 静态托管。CMS 路径可选接入飞书 BitTable API。
