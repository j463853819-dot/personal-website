# Personal Website Builder

> 一套面向 AI 助手的通用个人网站生成工作流。让非技术用户通过对话搭建个人网站，无需自己写代码。

对话式建站：回答几个关于你是谁、你有什么内容的问题，AI 帮你选风格、搭结构、填内容，生成一个可直接打开和部署的静态个人网站。如果需要持续更新而不碰代码，可选接入 CMS。

---

## 核心路径

整个工作流走的是一条「用户答 → AI 做 → 用户确认」的路：

```
身份访谈（你是谁）→ 结构确认（哪些板块、怎么排）
  → 选内容更新方式（默认纯静态，可选 CMS）
  → 选风格（6 种设计，各有模板）
  → 生成内容（模板填充 + 自检）
  → 本地交付 / 可选部署
  → 交付更新指南
```

### 关键设计决策

**Phase 1.5 是强制性步骤。** AI 不会猜你该放哪些板块——而是基于你的身份推荐一个结构，摊开给你看，等你确认了再往下走。

**默认顺序：Hero → 项目 → 文章 → 关于 → 联系。** 把访客最想看的内容（项目、文章）放前面，关于页放后面作为深度阅读者的背景信息。联系页永远在最后。

## 安装与使用

把 `personal-website` 文件夹放进你正在使用的 AI 工具所支持的本地技能、工作流或自定义指令目录中。

如果你的 AI 工具支持加载本地 skill/workflow，直接加载这个目录即可。如果不支持，也可以把 `SKILL.md` 作为主工作流说明，把 `references/` 和 `assets/` 作为配套资料与模板使用。

然后对你的 AI 助手说：

```text
帮我建一个个人网站。
```

或：

```text
Build me a personal portfolio site.
```

默认输出是 `dist/index.html` 和一份 `HOW_TO_UPDATE.md`。如果当前环境有部署工具，也可以继续部署到 GitHub Pages、Vercel、Netlify、CloudStudio 等静态托管平台。

## 六种设计风格

不搞「随便选个颜色」。每种风格有完整的 CSS 系统、配色方案、字体族和布局规则。第一版默认只选一种完整模板，不混搭、不临时改配色，目的是先稳定做出一个能用、好看的版本。第一版出来后，用户可以继续让 AI 调整配色、模块和细节。

| # | 风格 | 关键词 | 适合 |
|---|------|--------|------|
| 1 | 黑白极简 | 灰度滤镜、大留白 | 设计师、工程师 |
| 2 | 可爱手绘 | emoji、圆角、暖色 | 创意自由职业者 |
| 3 | 杂志编辑 | 衬线标题、印刷品质感 | 写作者、产品人 |
| 4 | 终端极客 | 黑底绿字、命令行美学 | 有代码作品的开发者 |
| 5 | 赛博霓虹 | 青紫品红三色发光 | 技术创作者 |
| 6 | 旅程叙事 | 大地色、时间线布局 | 想讲故事的创作者 |

## 内容更新方式

### 纯静态
默认路径。内容写在 HTML 里。要更新时，回来告诉 AI 改哪里，重新生成即可。适合大多数个人主页、作品集和低频更新场景。零依赖、零服务器、零密钥。

### 可选 CMS
适合每天或每周都更新文章、项目、资源列表的人。可以接入飞书多维表格等内容源，但必须通过安全的后端、Serverless Function、代理接口或构建脚本读取数据。

不要把飞书 `app_secret`、访问令牌或任何私密配置写进前端 HTML/JS。只配置飞书表格并不等于可以安全上线。

## 设计原则

1. **模板优先。** 不从零写 HTML。6 个成品模板是 CSS 的唯一权威来源。第一版先填内容，不急着改 CSS。
2. **首版硬约束。** 颜色不用自定义 hex——每种风格有预设调色板。字体族不可替换。响应式断点不可改。这样能更稳定地输出第一版；后续迭代再做个性化调整。
3. **自检不可跳过。** 部署前必须过 P0 清单：meta 标签、移动端适配、图片不撑破布局、CTA 在首屏、内部锚链接有效、中文网站无英文标签。
4. **交付的不只是页面。** 额外给一份 `HOW_TO_UPDATE.md`，告诉用户怎么改内容——无论是回来重新生成，还是通过安全 CMS 更新。

## 边界

以下场景明确不适合：
- 电商（购物车、支付、订单管理）
- 社区/论坛（用户注册、评论、实时聊天）
- 实时协作编辑
- 复杂的权限控制
- 百万级流量的生产应用

---

## 技术栈

纯 HTML + CSS + JavaScript。默认单文件，零框架，零构建。可部署到任何静态托管平台。CMS 路径可选，但需要安全的数据读取层，不能在前端暴露密钥。
